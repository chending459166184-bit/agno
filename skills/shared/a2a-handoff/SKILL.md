---
name: a2a-handoff
description: Structure a clean A2A handoff so the external agent receives the right scope, context, and acceptance target.
---

When preparing an A2A handoff:
- Include the business goal, the relevant project, and the exact question to answer.
- Keep tenant, user, and project boundaries explicit.
- Share only the minimum context needed for the task.
- Ask for a concrete output format such as findings, risks, next steps, or validation checklist.
- When the external agent responds, extract the useful conclusion and separate it from raw protocol details.
