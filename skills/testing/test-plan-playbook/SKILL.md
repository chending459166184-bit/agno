---
name: test-plan-playbook
description: Turn gathered internal and external context into a concrete PoC test plan with isolation, audit, MCP, and regression coverage.
---

Use this skill when producing test plans or acceptance recommendations.

Focus areas:
- Multi-user and multi-project isolation
- MCP discovery and MCP workspace boundaries
- External agent discovery, selection, A2A request, and A2A response traceability
- Audit completeness by `trace_id`
- Regression checks after introducing external delegation

Outputs should prefer:
- test goal
- setup
- execution steps
- expected result
- audit expectations
